ctxproxy

Context-aware proxy for Ollama Cloud — response caching, token monitoring, adaptive compression, and prefix caching for OpenAI-compatible APIs.

GitHub → Quick Start

Architecture

Client (Hermes, Claude Code, Codex, ...)
│
▼
ctxproxy:11438
│
├── Response cache (exact match, TTL 120s)
├── Token monitoring (warn at 80%, critical at 95%)
├── Adaptive compression (sliding window + similarity)
└── Prefix caching (content-addressed KV, SHA-256)
│
▼
Ollama Cloud:443
──────────────────────────────────
~505K token limit (infrastructure)
No native prompt caching
No context monitoring

Features

⚡

Response Caching

Exact-match cache with configurable TTL. SQLite-backed, survives restarts. Avoids re-calling the API for identical requests.

📊

Token Monitoring

Tracks every request's token usage. Warns at 80% of the context limit, critical at 95%. Returns a proper 400 before Ollama Cloud's hard limit.

📦

Adaptive Compression

When estimated tokens exceed 70% of the limit, automatically compresses: keeps system message, last N messages, and semantically similar messages.

🔑

Prefix Caching

Content-addressed KV cache for the static prefix (system + initial exchanges). SHA-256 hashing. Logs hits and misses for observability.

📈

Stats Endpoint

GET /stats returns call count, total tokens, cache entries, prefix entries, compression count, and recent events.

🔌

Zero Changes

Drop-in proxy — no modifications to your client code. Works with any OpenAI-compatible client. Single file, no heavy dependencies.

Quick Start

# Install
pip install aiohttp pyyaml

# Start the proxy
OLLAMA_API_KEY=your_key_here python ctxproxy.py

# Point your client to http://127.0.0.1:11438/v1
# For Hermes:
hermes --provider ctxproxy
# Check stats
curl http://127.0.0.1:11438/stats

# Response:
{
  "calls": 42,
  "total_tokens": 147176,
  "cache_entries": 5,
  "prefix_entries": 3,
  "compressions": 1
}

Bugs Found & Fixed

1. SQLite schema drift

Adding a compressed column to an existing DB caused a 500 error. Fixed with CREATE TABLE IF NOT EXISTS and migration checks.

2. self.cache.db_path vs self.db_path

Copy-paste error from when ResponseCache was a property of CtxProxy. Fixed by using self.db_path directly.

3. Streaming breaks resp.json()

SSE streams return text/event-stream — calling resp.json() fails. Fixed by checking body.stream before reading.

4. Port collision with local Ollama

Port 11435 was already used by a local Ollama GPU instance. Default changed to 11438.
